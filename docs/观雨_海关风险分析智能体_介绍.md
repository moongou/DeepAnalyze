## 一、产品概述

**"观雨"** 是一款专为中国海关风险管理和风险防控打造的 AI 智能体。它精通 Python 与 R 语言，基于 [DeepAnalyze](https://github.com/ruc-datalab/DeepAnalyze) 大模型引擎（首个面向自主数据科学的智能体大语言模型），能够**自主完成**从数据探索、统计分析、关联挖掘到专业报告生成的全流程工作——**无需人工编写一行代码**。

> 💡 **一句话介绍**：上传进出口业务数据，观雨帮你深度洞察隐藏的风险线索，并一键生成分析师级专业报告。

### 为什么叫"观雨"？

"观雨知天下"——通过观察数据中的"风雨"信号，洞察进出口贸易中的暗流与风险。配套的特色功能也延续了这一命名风格：

| 功能名 | 寓意 |
|--------|------|
| **风调雨顺** | 分析过程中的实时指导与调控 |
| **雨途斩棘录** | 错误修正知识库，越用越智能 |

## 四、智能分析工作流

观雨遵循严格的结构化分析流程，确保每一步都有据可依：

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  数据探索  │ →  │  分析规划  │ →  │  智能执行  │ →  │  深度洞察  │ →  │  报告交付  │
│          │    │          │    │          │    │          │    │          │
│ 自动识别   │    │ 海关业务   │    │ Analyze   │    │ 多角度    │    │ PDF      │
│ 字段类型   │    │ 逻辑制定   │    │ → Code   │    │ 逻辑推理  │    │ DOCX     │
│ 建立数据   │    │ 分析角度   │    │ → Execute│    │ 穿透式    │    │ PPTX     │
│ 字典      │    │ 构建任务树  │    │ 自我纠错  │    │ 关联挖掘  │    │ 交付检查  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
```

### 🤖 Agentic 执行范式

观雨基于 DeepAnalyze 的智能体标签系统运作：

| 标签 | 作用 | 说明 |
|------|------|------|
| `<Analyze>` | 任务规划与推理 | 预测推理、假设验证、结果解释 |
| `<Understand>` | 数据探索 | 主动探索数据源结构与语义 |
| `<Code>` | 代码生成 | 生成可执行的 Python/R 代码 |
| `<Execute>` | 代码执行 | 系统自动注入执行结果 |
| `<Ask>` | 交互询问 | 交互模式下主动向用户确认 |
| `<Answer>` | 结论输出 | 输出最终分析结论与建议 |

### 📋 两种分析模式

| 模式 | 适用场景 | 特点 |
|------|----------|------|
| **🤝 交互式分析** | 需要精细控制分析方向 | 展示计划 → 用户选择 → 逐项执行 |
| **🤖 全程代理分析** | 对结果有明确预期 | 自主执行全部任务树 → 一次性交付 |

## 五、三级分析策略

根据任务复杂度和分析需求，灵活选择分析深度：

### 🎯 聚焦诉求（温度 ≈ 0.2）

> **严格遵循用户指令，直击要点，不做多余发散。**

- 仅针对用户直接提出的问题分析
- 简洁高效，快速给出结论
- 适用于：明确的单一分析需求，如"某企业价格偏离度分析"

### ⚖️ 适度扩展（温度 ≈ 0.4）

> **满足核心需求，适量关联，保持分寸。**

- 基于数据表现进行适量关联分析
- 提供背景信息和浅层风险提示
- 适用于：需要一定上下文的分析，如"某行业进口整体态势"

### 🔮 广泛延展（温度 ≈ 0.6）

> **深度发散，充分挖掘数据间的潜在联系。**

- 结合海关业务逻辑全方位风险预判
- 趋势分析、关联挖掘、前瞻性建议
- 适用于：深度研究型任务，如"某区域通关异常综合研判"

## 六、穿透式关系图谱分析

观雨具备基于图论和关系型数据库思维的**穿透式分析能力**，可从层层关系中锁定幕后实质性主体。

### 🏢 实体识别

从数据中自动识别核心实体节点：

- 🏭 经营企业 / 收发货人 / 货主单位
- 👤 法定代表人 / 实际控制人
- 📦 货物 / HS编码 / 商品类别
- 🚢 运输工具 / 运输路线
- 🏦 银行账户 / 资金流向
- 📝 报关单位 / 代理企业

### 🔍 六大穿透策略

| 策略 | 说明 |
|------|------|
| **直接关系分析** | 一度关联：直接交易、申报、代理关系 |
| **间接关系分析** | 二度及以上：通过中间节点的隐性关系 |
| **路径分析** | 任意两个实体间的最短路径与所有路径 |
| **社区发现** | 识别紧密关联的实体群组（利益共同体） |
| **中心性分析** | 度中心性 / 介数中心性，定位核心控制节点 |
| **异常模式识别** | 星形结构（壳公司）、链式结构（层层代理） |

### 🛠 技术支撑

```python
import networkx as nx

# 构建关系图谱
G = nx.Graph()
G.add_node("企业A", type="enterprise", risk_level="high")
G.add_node("个人B", type="person", role="法定代表人")
G.add_edge("企业A", "个人B", relation="法定代表人", weight=1.0)

# 穿透分析
degree_centrality = nx.degree_centrality(G)      # 度中心性
betweenness = nx.betweenness_centrality(G)        # 介数中心性
communities = nx.community.greedy_modularity_communities(G)  # 社区发现
```

支持 **Neo4j 图数据库集成**，当数据规模较大时可无缝切换到图数据库引擎。

### ✨ 四大特色功能

#### 🌧 风调雨顺 —— 过程指导
在分析进行中，随时注入新的需求、方法或条件。这些信息将与当前任务结合，动态指导智能体的下一步动作——让分析方向始终可控。

#### 📖 雨途斩棘录 —— 错误修正知识库
- 每次遇到错误并解决后，自动记录解决方案

- 自动识别代码执行错误类型
- 生成修复方案并重试（最多3次）

#### 🚫 死循环检测
- 智能检测分析死循环（相同逻辑≥3次/相同代码≥2次无新洞察）
- 强制跳出 → 记录问题 → 继续下一任务
- 确保分析流程永不卡死

## 八、九大分析原则

观雨遵循**九大核心原则**，确保每次分析的质量与可靠性：

### 原则一：🔄 自我纠错与错误学习
自动识别错误类型（数据缺失、类型错误、维度不匹配、编码问题），生成修复方案重试。每次修复记录到雨途斩棘录，永不重蹈覆辙。

### 原则二：🧪 先短代码测试可行性
执行复杂分析前，先用 2-5 行代码验证关键假设（字段名、数据类型、文件路径），确认可行后再执行完整分析。

### 原则三：📚 数据字典优先
**分析首步必须建立数据字典**：字段名、数据类型、非空率、唯一值数量、代表性样本。后续所有分析严格引用字典中的实际字段名。

### 原则四：🧩 大任务拆分为小任务
将复杂分析目标分解为结构化任务树，按依赖关系编排执行顺序，支持并行处理无依赖任务，共性逻辑提取为公共函数。

### 原则五：📋 双模分析
支持交互式分析（用户选择执行）与全程代理分析（自主完成全部），灵活适配不同使用场景。

### 原则六：📝 输出规范
所有输出使用简体中文，报告支持 PDF/DOCX/PPTX 三格式，海关风格模板排版，图表统一使用专业主题。

### 原则七：🔐 数据类型安全
自动检测列类型，安全执行类型转换（`pd.to_numeric(errors='coerce')`），对异常类型给出警告并自动修复。

### 原则八：⚡ 高效处理与代码复用
递进式成果累积（1→12→123→1234），禁止从头开始。复用已有中间结果，共性逻辑提取为公共函数。

### 原则九：✅ 最终交付物检查
- 报告格式完整性（PDF/DOCX/PPTX 是否全部生成）
- 分析目标覆盖率（所有需求是否已满足）
- 风调雨顺检查（额外需求是否已体现）
- 图表引用正确性（所有图表是否正确嵌入）
- 报告质量检查（每个分析点是否包含"图表+文字+数据"三要素）

## 九、专业报告交付

### 📑 报告结构规范

每份分析报告严格遵循三段式结构：

#### 第一部分：分析思路
- 阐述分析的背景和原因
- 描述推理逻辑和将要采取的步骤
- 明确分析目标与预期产出

#### 第二部分：分析主体内容
每个分析维度包含四要素：
1. **现状描述** — 观察到的数据事实
2. **推理推导** — 基于数据的逻辑推理
3. **阶段性结论** — 初步发现
4. **可视化图表** — 紧跟观点的数据支撑

> ⚠️ **核心要求**：严禁"直接得结论"，必须体现"数据 → 推理 → 结论"的层次感。

#### 第三部分：分析小结
- 核心发现汇总
- 风险等级评估
- 针对性建议
- 汇总表格（关键发现 + 风险等级）

### 📦 三格式交付

| 格式 | 特点 | 排版规范 |
|------|------|----------|
| **📕 PDF** | 海关风格模板 | 黑体标题(16-22pt) + 仿宋正文(10-12pt) + 楷体强调 |
| **📘 DOCX** | 支持二次编辑 | 结构化 Word 文档，保留完整格式 |
| **📙 PPTX** | 适合汇报展示 | 演示文稿，图表+结论提炼 |

### ✅ 质量红线

```
🚫 严禁空壳报告    — 不允许 "[详细方法描述...]" 等占位文本
🚫 严禁结论堆砌    — 必须有推理过程和数据证据
🚫 严禁图表堆放    — 图表必须与对应观点紧密结合
✅ 每个分析点必须   — "有图、有话、有数据"
```

## 十一、典型应用场景

### 🕵️ 场景一：走私违规线索排查

**任务**：分析进出口企业申报数据，识别潜在走私与违规申报线索。

**分析维度**：
- 价格偏离分析（同类商品价格对比）
- 申报行为异常检测（频率、数量突变）
- 历史违规记录关联

**产出**：风险企业排名 + 可疑行为详述 + 推理依据链条

---

### 💰 场景二：逃税风险画像

**任务**：构建逃税风险企业画像，量化风险评分。

**三维交叉分析**：
1. 低报价格维度 — 与同类商品均价对比
2. 伪报原产地维度 — 产地与运输路线逻辑校验
3. HS编码归类偏差维度 — 品名与编码匹配度分析

**产出**：风险评分排名 + 特征工程报告 + SHAP 解释

---

### 🔗 场景三：利益关联网络穿透

**任务**：基于企业关系图谱，发现隐蔽关系结构。

**穿透路径**：
- 壳公司识别 → 星形结构检测
- 关联交易检测 → 资金流向追踪
- 层层代理穿透 → 链式结构分析

**产出**：关系图谱可视化 + 核心节点报告 + 利益集团识别

---

### 📈 场景四：行业趋势与异常预警

**任务**：某行业/商品/区域进出口态势分析。

**分析内容**：
- 时序趋势识别（月度/季度/年度）
- 周期模式发现与季节性分析
- 异常波动检测与实时预警

**产出**：趋势分析报告 + 异常预警清单 + 前瞻性建议

## 十二、快速开始

### 📥 环境准备

```bash
# 创建 Python 环境
conda create -n deepanalyze python=3.12 -y
conda activate deepanalyze
pip install -r requirements.txt
```

### 🤖 模型下载与部署

从 [HuggingFace](https://huggingface.co/RUC-DataLab/DeepAnalyze-8B) 或 [ModelScope](https://www.modelscope.cn/models/RUC-DataLab/DeepAnalyze-8B) 下载模型，然后：

